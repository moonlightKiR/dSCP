\section{Diseño Experimental}

Para validar la capacidad de generalización del sistema y detectar posibles sesgos, se ha diseñado el siguiente protocolo de pruebas:

\begin{itemize}
    \item \textbf{Estimador de Error:} Se empleará una estrategia de Holdout con una partición del \verb|80%| para entrenamiento y \verb|20%| para test.
    Adicionalmente, se realizará una \textit{Validación Cruzada (k-fold)} durante la fase de optimización con \textit{Optuna} para garantizar que el rendimiento es independiente de la partición de datos.
    \item \textbf{Métricas de Evaluación:}
    \begin{itemize}
        \item \textit{Matriz de Confusión:} Fundamental para cuantificar los falsos positivos.
        \item \textit{F1-Score y AUC-ROC:} Para medir la precisión y la capacidad de discriminación del modelo en un entorno de clasificación binaria.
    \end{itemize}
\end{itemize}
